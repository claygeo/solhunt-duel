# `_ops/` — internal operational artifacts

Files in this directory are **operational templates and drafts**, not part of the public Solhunt-Duel portfolio. Recruiters and engineers evaluating the project should look in `docs/` instead.

The `_` prefix sorts this directory below `docs/` alphabetically and signals "internal."

## Contents

- [`WEEKLY-CADENCE-TEMPLATE.md`](WEEKLY-CADENCE-TEMPLATE.md) — Friday weekly post template (LinkedIn / Twitter / Substack rotation), used internally to coordinate the recurring drumbeat.
- [`SUBSTACK-POST-1-DRAFT.md`](SUBSTACK-POST-1-DRAFT.md) — draft of Substack post #1 (67%/13% gap origin story). Pre-publish, requires final operator pass + /codex outside-voice review before going live.

## Why these are here, not in `docs/`

[/codex outside-voice meta-review, 2026-04-28] flagged that operational artifacts in the public `docs/` folder dilute the portfolio signal. A recruiter reading `docs/SUBSTACK-POST-1-DRAFT.md` sees the meta-game ("I'm pre-positioning"). Self-aware framing in a draft is fine; *publishing the draft of the framing* is not.

These files moved here to keep `docs/` strictly portfolio: architecture, proof of correctness, case studies, plans for review. The weekly-cadence playbook and pre-publish post drafts are tools, not portfolio.

## The blocker — re-auth VPS claude

```
$ ssh -p 2222 root@77.42.83.22 "claude -p --model claude-opus-4-7 --output-format json 'hi'"
{"is_error":true,"result":"Not logged in · Please run /login","total_cost_usd":0,...}
```

`/root/.claude/.credentials*` doesn't exist. Auth must have expired or been wiped sometime in the last 8 days.

## How to unblock (do this when you wake up)

1. SSH to VPS:
   ```
   ssh -p 2222 root@77.42.83.22
   ```

2. Run `claude` (no flags):
   ```
   claude
   ```
   It'll print an OAuth URL.

3. Open URL on your laptop, complete the Anthropic OAuth flow.

4. Confirm token saved:
   ```
   ls -la /root/.claude/.credentials*
   ```
   You should see a file. Then `exit` SSH.

5. Tell me to launch the scan, OR run it yourself:
   ```
   ssh -p 2222 root@77.42.83.22 "cd /root/solhunt && tmux new-session -d -s solhunt-live 'bash scripts/meta-live-scan.sh'"
   ```

## How to check progress while it's running

```
# tail the meta-runner log
ssh -p 2222 root@77.42.83.22 "tail -f /root/solhunt/findings/meta-run-*/run.log"

# or attach to the tmux session live
ssh -p 2222 -t root@77.42.83.22 "tmux attach -t solhunt-live"
# (detach with Ctrl-b then d)
```

## Where results land

```
/root/solhunt/findings/
  meta-run-<ts>/
    run.log               # high-level meta-runner log
    1-RepoDriverLogic.log # full per-scan stdout/stderr
    1-RepoDriverLogic.exit
    STATUS                # FOUND | NO_FIND | RATE_LIMITED | ERROR_BUDGET | TOTAL_TIMEOUT

  <iso-ts>-RepoDriver/    # bundle from solhunt itself when --via-claude-cli used
    report.json           # structured agent verdict
    Exploit.t.sol         # Foundry test (if claude wrote one)
    README.md             # human-review checklist — READ THIS FIRST
```

If `STATUS` says `FOUND`, **DO NOT submit to Immunefi without re-running the exploit on a fresh fork.** The README in the finding bundle has the full review checklist.
